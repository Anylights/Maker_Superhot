## 总体策略

用 AI 图片生成工具（`generate_image` / `batch_generate_images`）逐张生成瓦片。
核心挑战是**风格一致性**和**边缘可拼接**，通过以下三个手段保证。

## Prompt 设计

### 统一前缀（所有瓦片共享）

```
俯视角[主题]tilemap瓦片,像素风格,统一[主色调]色调,128x128正方形单个瓦片。
```

示例：
```
俯视角森林草地tilemap瓦片,像素风格,统一绿色草地色调,128x128正方形单个瓦片。
```

### 统一后缀

```
俯视视角,无阴影,平铺纹理风格
```

### 颜色词锁定（铁律）

**同一元素在所有 prompt 中必须使用完全相同的颜色描述词。**

先定义一套颜色词表，全套瓦片严格复用：

```
草地 → "绿色草地"      （不要有的写"翠绿"有的写"深绿"有的写"青绿"）
海水 → "深蓝色海水"    （不要有的写"蔚蓝"有的写"湛蓝"有的写"靛蓝"）
路面 → "土黄色泥土路面" （不要有的写"棕色"有的写"米色"有的写"沙色"）
沙滩 → "黄褐色沙滩"
```

边缘/角落瓦片描述中引用这些元素时，必须原样复用颜色词：

```
✅ "上半部分是绿色草地,下半部分是深蓝色海水"
❌ "上半部分是青翠的草坪,下半部分是蔚蓝的水面"
```

### 中间描述：精确方向性

**边缘瓦片**必须明确描述"哪半是什么"：

- road_edge_N: "上半部分是绿色草地,下半部分是土黄色泥土路面,中间有自然的草地到路面过渡边缘线"
- road_edge_W: "左半部分是土黄色泥土路面,右半部分是绿色草地,中间有自然的垂直过渡边缘线"

**角落瓦片**必须明确"哪个角是什么"：

- road_corner_NW: "左上角是土黄色泥土路面,右上角和下方是绿色草地,路面从左上角弯曲过渡到草地,转角边缘圆滑自然"
- road_inner_NW: "大部分是土黄色泥土路面,仅在左上角有一小块圆弧形绿色草地突出"

**装饰瓦片**说明元素占比和底色：

- "绿色草地上的灌木丛,几个深绿色小灌木圆形簇生分布在绿色草地上,作为装饰元素,草地部分能无缝拼接"

### 不要做的事

- 不要在 prompt 中描述多个瓦片（如"一套瓦片"、"左边是A右边是B的过渡组合"）
- 不要写模糊的方向（如"边缘有过渡"），要明确"上/下/左/右/左上/右下"
- 不要使用3D透视相关词汇（如"立体"、"高光"、"阴影"）
- **不要对同一元素使用不同的颜色描述词**（铁律）

## 风格一致性三板斧

| 手段 | 作用 | 用法 |
|------|------|------|
| **统一颜色词** | 语义层面锁定每种元素的颜色 | 定义颜色词表，所有 prompt 严格复用 |
| **固定 seed** | 让 AI 生成风格趋于稳定 | 所有瓦片使用同一个 seed（如 42） |
| **reference_image** | 视觉锚点，拉齐色彩/画风 | 第 1 张生成后，后续瓦片都引用它作为参考图 |

三者缺一不可。颜色词管语义一致，seed 管生成倾向稳定，reference_image 管视觉色板锚定。

### 执行顺序

1. 先定义**颜色词表**（草地=绿色草地，海水=深蓝色海水……）
2. 单独生成 1 张**最基础的瓦片**（如 grass_base1），不带 reference
3. 检查满意后，将其作为 reference_image 传给后续所有瓦片
4. 后续瓦片用 `batch_generate_images` 批量生成（每批 2-4 张避免超时）

## 分批生成计划

推荐分 5 批，从基础到细节：

| 批次 | 内容 | 张数 | 说明 |
|------|------|------|------|
| 1 | 草地基础 x4 + 路面基础 x2 | 6 | 先生成 grass_base1，再批量生成其余 5 张 |
| 2 | 草路边缘 x4 + 外角 x4 | 8 | 分两批各 4 张 |
| 3 | 路面内角 x4 + 森林 x4 | 8 | 分两批各 4 张 |
| 4 | 森林边缘 x4 + 装饰 x4 | 8 | 分两批各 4 张 |
| 5 | 地形特征 x5 | 5 | 一批或分两批 |

## 批次执行细节

### 第 1 张（锚点）

```python
generate_image(
    prompt="[统一前缀]绿色草地地面,均匀分布的绿色草纹理,可无缝平铺。[统一后缀]",
    name="grass_base1",
    target_size="128x128",
    aspect_ratio="1:1",
    seed=42
)
```

### 后续批量

```python
batch_generate_images(images=[
    {
        "prompt": "[统一前缀]...[统一后缀]",
        "name": "grass_base2",
        "target_size": "128x128",
        "aspect_ratio": "1:1",
        "seed": 42,
        "reference_images": ["assets/image/grass_base1_TIMESTAMP.png"]
    },
    # ... 更多瓦片
])
```

### 超时处理

- `batch_generate_images` 超过 4 张时容易超时
- 首选方案：拆成每批 2-4 张
- 备选方案：逐张用 `generate_image` 生成

## 验收检查

生成完所有瓦片后：

1. **检查总数**：确认生成了计划中的全部瓦片
2. **检查尺寸**：所有图片必须是 128x128（或项目要求的统一尺寸）
3. **颜色一致性**：对比同元素在不同瓦片中的颜色，确认无色差
4. **复制到目标目录**：从 `assets/image/` 复制到 `assets/Textures/tiles/`，去掉时间戳后缀
5. **更新代码**：更新编辑器的 tileDefs 数组，按分类组织
6. **构建测试**：调用 build 工具验证图片能正确加载
