## 核心原则

1. **每张图只含一个瓦片** — 禁止一张图包含多个瓦片
2. **所有瓦片必须正方形** — 统一尺寸（推荐 128x128）
3. **相邻瓦片边缘必须无缝拼接** — 相接的两条边在颜色和纹理上平滑过渡
4. **风格统一** — 画风、分辨率、色彩、光照方向在全套瓦片中保持一致
5. **同元素同颜色** — 所有瓦片中相同的元素，颜色必须完全一致（见下方铁律）

## 颜色一致性铁律

**所有瓦片中出现的相同元素，颜色必须完全一致。这是最基本的拼接前提。**

### 规则

同一种地形/元素无论出现在哪张瓦片中，其颜色必须是同一个颜色，不能有色差：

| 元素 | 要求 |
|------|------|
| 草地 | grass_base、road_edge 里的草地部分、ocean_edge 里的草地部分、corner 里的草地部分 → **同一个绿色** |
| 海水 | ocean_deep、ocean_edge 里的海水部分、ocean_corner 里的海水部分、ocean_inner 里的海水部分 → **同一个蓝色** |
| 路面 | road_base、road_edge 里的路面部分、road_corner 里的路面部分 → **同一个土黄色** |
| 沙滩 | sand_base、过渡带中的沙滩色 → **同一个黄褐色** |

### 反面案例

```
❌ grass_base 是翠绿色，但 ocean_edge_N 上半部分的草地偏墨绿
   → 拼在一起时草地区域出现明显色块断裂

❌ ocean_deep 是深蓝色，但 ocean_corner_NW 的海水部分偏紫蓝
   → 海面区域拼接后出现色差补丁

❌ road_base 是暖黄土色，但 road_edge_N 的路面部分偏冷灰
   → 路面看起来像两种不同材质
```

### Prompt 写法保障

在 AI 生成时，用**明确的颜色词**锁定每种元素的颜色，且全套瓦片的 prompt 中同一元素使用完全相同的颜色描述：

```
✅ 所有 prompt 中草地统一写 "绿色草地"（不要有的写"翠绿"有的写"深绿"有的写"青绿"）
✅ 所有 prompt 中海水统一写 "深蓝色海水"（不要有的写"蔚蓝"有的写"湛蓝"有的写"靛蓝"）
✅ 所有 prompt 中路面统一写 "土黄色泥土路面"
```

配合 seed + reference_image 进一步约束，三者缺一不可：
1. **Prompt 颜色词一致** — 语义层面锁定
2. **固定 seed** — 生成倾向稳定
3. **reference_image** — 视觉层面锚定色板

## 16-Tile 过渡系统

两种地形（A 和 B）之间的完整过渡需要 16 张瓦片：

| 类型 | 数量 | 说明 |
|------|------|------|
| A 基础 | 2-4 | 纯地形 A（多张变体避免重复感） |
| B 基础 | 1-2 | 纯地形 B |
| 边缘 | 4 | 上(N)、下(S)、左(W)、右(E) — 一半 A 一半 B 的水平/垂直分割 |
| 外角 | 4 | NW、NE、SW、SE — 某一角是 B，其余是 A |
| 内角 | 4 | NW、NE、SW、SE — 大部分是 B，某一角有 A 弧形突出 |

外角示例（road_corner_NW）: 左上角是 B，其余 A
内角示例（road_inner_NW）: 左上角小块弧形 A，其余 B

## 标准瓦片分类

推荐的分类体系（35 张完整套装为例）：

- grass/ — 草地基础 x4（变体）
- road/ — 路面基础 x2 + 边缘 x4 + 外角 x4 + 内角 x4 = 14
- forest/ — 浓密森林 + 稀疏森林 + 单棵树 x2 + 森林边缘 x4 = 8
- decor/ — 灌木丛、树桩、花丛、小石头 = 4
- terrain/ — 悬崖顶/壁、水塘、蘑菇、路牌 = 5

可根据游戏需求增减，但每组过渡至少需要：基础 + 4 边缘 + 4 外角 + 4 内角。

## 边缘匹配铁律

| 规则 | 说明 |
|------|------|
| 水平边缘 | road_edge_N 的下边 = road_base 的上边；road_edge_N 的上边 = grass_base 的下边 |
| 垂直边缘 | road_edge_W 的右边 = grass_base 的左边；road_edge_W 的左边 = road_base 的右边 |
| 角落衔接 | road_corner_NW 的上边 = road_edge_N 能接的边；左边 = road_edge_W 能接的边 |
| 基础瓦片 | 多个变体之间的四条边都能互相拼接（颜色、纹理连续） |

## 风格一致性要求

以下要素必须在全套瓦片中统一：

- **视角**：统一俯视角（top-down），不混入 45 度或侧视角
- **光照**：统一无阴影或统一光照方向
- **色调**：使用统一色板（如草地统一为同一绿色系）
- **像素密度**：统一分辨率，不可混入高清和像素风
- **纹理风格**：统一平铺纹理（tileable texture），不可有画框感
